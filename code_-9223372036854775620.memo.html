Manche methoden sind aus 2 Gründen schwierig:

1. Ihr Name suggeriert etwas anderes, als das, was sie wirklich tut (allerdings ist der Name konsistent - d.h. andere Bibliotheken verwenden den gleichen Namen in gleicher - inunitiver - Absicht)

2. Ihr Gebrauch ist optional, d.h. das Verstehen ist für einen Anfänger nicht notwendig

 

Gefahr:

Wenn ein Anwender bei der Suche einer Funktion durch einen verwechselbaren Funktionsnamen gefunden wurde, besteht die Gefahr, dass die Funktion für die richtige gehalten wird.

Wegen der Synonymfeindlichkeit wird der Anwender möglicherweise annehmen, die Funktion einfach nur falsch zu verwenden oder einen Bug gefunden zu haben.

 

Beispiel:

String-Funktion "reserve": "Increases the capacity" nicht intuitiv + Fehlermeldung bei Verwendung

"Assertion failed : static_cast<TStringPos>(pos) < static_cast<TStringPos>(length(me)) was: 0 >= 0 (Trying to access an element behind the last one!)"
